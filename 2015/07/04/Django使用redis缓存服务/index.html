Django使用redis缓存服务

By djskl

发表于 2015-07-04

django中的缓存系统是比较完善的，不但支持MemcachedCache、redis、数据库、本地内存，本地文件，还有开发用的DummyCache，而且还可以通过在http头加入相关信息来设置下游缓存(比如squid，浏览器缓存)。从缓存粒度来说，支持整站、view、template标签等多级别的缓存服务。

本文主要介绍基于redis的缓存服务。

1、安装

pip install django-redis

2、配置

CACHES = {
  ”default”: {
    ”BACKEND”: “django_redis.cache.RedisCache”,
    ”LOCATION”: “redis://127.0.0.1:6379/1”,
    ”OPTIONS”: {
        ”CLIENT_CLASS”: “django_redis.client.DefaultClient”,
    }
  }
}

3、使用

cache_page(timeout)可以缓存当前被修饰的函数(index)的返回值，看一下cache_page的说明文档：

Decorator for views that tries getting the page from the cache and populates the cache if the page isn’t in the cache yet.

The cache is keyed by the URL and some data from the headers.Additionally there is the key prefix that is used to distinguish different cache areas in a multi-site setup. You could use the sites.get_current_site().domain, for example, as that is unique across a Django project.

Additionally, all headers from the response’s Vary header will be taken into account on caching – just like the middleware does.

装饰器的解析流程如下：
@cache_page(600)
    ||
index = cache_page(600)(index)
    ||
index = decorator_from_middleware_with_args(CacheMiddleware)(600)(index)
    ||
index = make_middleware_decorator(CacheMiddleware)(600)(index)
    ||
index = _make_decorator(600)(index)
    ||
index = _decorator(index)
    ||
_wrapped_view(request, *args, **kwargs)

_decorator是一个通用的中间件的装饰器，有点像基于中间件的策略模式，在这里使用的中间件是CacheMiddleware，视图缓存的存取更新都是通过这个中间件实现的：
