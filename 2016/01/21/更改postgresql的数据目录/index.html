更改postgresql的数据目录

By djskl

发表于 2016-01-21

生产环境的postgresql数据出了问题，于是就想把整个数据库拷到本地来调试一下。

1、找到生产环境的postgresql的data目录，在命令行下连接上数据库，然后执行show data_directory就可以看到了，在fedora20上是/var/lib/pgsql/data(通过yum安装)。

2、转到本地，停掉本地的数据库，找到本地的data目录重命名为data.old暂做备份，然后把生产环境的data目录拷到一个可以放的下它的地方，比如/new/path/folder/data。注意权限，保证这个目录必须是postgres用户可读可写的。

3、建立软连接并重启数据库就可以了：

ln -s /new/path/folder/data /var/lib/pgsql/data

附：Fedora上yum安装postgresql的目录(转自:Just installed postgresql what path is it stored?)

  • Executables : /usr/bin
  • Libraries : /usr/lib
  • Documentation : /usr/share/doc/postgresql-x.y.z , /usr/share/doc/postgresql-x.y.z/contrib
  • Contrib : /usr/share/pgsql/contrib
  • Data : /var/lib/pgsql/data
  • Backup area : /var/lib/pgsql/backup
  • Templates : /usr/share/pgsql
  • Procedural Languages : /usr/lib/pgsql
  • Development Headers : /usr/include/pgsql
  • Other shared data : /usr/share/pgsql
  • Regression tests : /usr/lib/pgsql/test/regress (in the -test package)
  • Documentation SGML : `/usr/share/doc/postgresql-docs-x.y.z
